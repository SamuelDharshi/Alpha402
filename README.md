## 🛡️ 0G-Native Verifiable Agent Orchestration

**Alpha402** is a decentralized multi-agent protocol built to anchor AI reasoning and DeFi execution into the **0G Labs ecosystem**. By treating 0G as the foundational trust layer, we have moved the entire lifecycle of an autonomous agent—from intent parsing to risk scoring—into a cryptographically verifiable environment.

---

## 🧠 0G Compute: Verifiable Intelligence Layer (TEE)

The core "brain" of Alpha402 is decentralized via the **0G Compute Network**, specifically leveraging **Trusted Execution Environments (TEE)** to ensure that AI reasoning is tamper-proof and independently verifiable.

### 1. TEE-Verified Broker Orchestration
Instead of simple API calls, we utilize the **Broker-Mode** architecture in the `ZGComputeNetworkBroker`. This ensures that every risk score generated by the **Risk Agent** is cryptographically signed by a verified TEE node.
*   **Signature Verification:** Every inference response is passed through the `processResponse` validation loop. This checks the provider's EIP-712 signature against the known TEE enclave address, ensuring the model ran exactly as expected without external manipulation.
*   **Decentralized Provider Selection:** The system dynamically fetches service metadata from the 0G Compute network, identifying providers that offer performance-based quality guarantees on the Galileo testnet.

### 2. 0G Settlement Ledger Integration
Alpha402 implements a programmatic **Economic Metering** system using the 0G Settlement Ledger.
*   **Auto-Funding Logic:** Before an agent swarm is deployed, the orchestrator queries the `unsettledFee` balance of the selected provider.
*   **OG Token Flow:** If the balance is insufficient, the system programmatically triggers a ledger top-up in **OG tokens**, ensuring that the compute pipeline remains continuous and fair.

## 💾 0G Storage: Immutable Provenance Engine

Alpha402 leverages **0G Storage** to solve the "Memory Gap" in autonomous agents. Every message, decision, and transcript is persisted to the decentralized network, creating a permanent, audit-ready history.

### 1. Merkle-Tree Data Construction
We utilize the `@0glabs/0g-ts-sdk` to transform ephemeral agent messages into structured, Merkle-tree compatible segments.
*   **Segment Preparation:** Data is chunked into fixed-size binary blobs, which are then hashed to generate a unique **Merkle Root (CID)**.
*   **Flow Contract Interaction:** The Merkle root is registered on the 0G Flow contract, creating a permanent on-chain reference to the off-chain data stored across the node network.

### 2. Redundant Replication & Failover
To ensure absolute data availability, Alpha402 implements a multi-indexer storage strategy.
*   **Parallel Uploads:** Agent transcripts are simultaneously submitted to multiple 0G Indexers (Standard and Turbo).
*   **Indexer Recovery:** We developed a custom failover handler for **503 (Congestion)** errors, automatically re-routing Merkle-tree submissions to alternative indexers.
